## Spacing Scale

SwiftUI `.padding()` values used throughout the project map to multiples of 4. This phase adopts the same values already established in `BezelView`, `GistSettingsSection`, and `DocsetSettingsSection`.

| Token | Value | SwiftUI Expression | Usage |
|-------|-------|--------------------|-------|
| xs | 4pt | `.padding(4)` | Icon-to-label gaps, footnote spacing |
| sm | 8pt | `.padding(8)` | Compact spacing between related elements |
| md | 16pt | `.padding(16)` | Standard element padding (TextField, form rows) |
| lg | 24pt | `.padding(24)` | Nag panel inner padding (matches RESEARCH.md sketch) |
| xl | 32pt | `.padding(32)` | Not used in this phase |
| 2xl | 48pt | `.padding(48)` | Not used in this phase |
| 3xl | 64pt | `.padding(64)` | Not used in this phase |

Exceptions:
- LicenseNagView VStack spacing: 16pt (matches RESEARCH.md sketch: `VStack(spacing: 16)`)
- HStack button spacing inside nag: 8pt (`HStack(spacing: 8)`) — standard scale; RESEARCH.md sketch used 12pt which is non-standard
- LicenseNagView frame width: 380pt fixed (matches RESEARCH.md sketch: `.frame(width: 380)`)

**Source:** RESEARCH.md Pattern 2 (LicenseNagView structure sketch); BezelView.swift (existing spacing patterns)

## Typography

System font (SF Pro) at system-defined sizes. No custom typefaces. All sizes are SwiftUI semantic sizes, not numeric pixel values — they scale with Dynamic Type on macOS.

| Role | SwiftUI Modifier | Approximate pt | Weight | Line Height |
|------|-----------------|----------------|--------|-------------|
| Body | `.font(.body)` | 13pt | Regular (400) | system default (~1.4) |
| Label / secondary | `.font(.callout)` | 13pt | Regular (400) | system default |
| Footnote / hint / inline error | `.font(.footnote)` | 11pt | Regular (400) | system default |
| Heading (nag title) | `.font(.title2.bold())` | 22pt | Bold (700) | system default (~1.2) |

**Constraints:**
- Maximum 2 weights in any single surface: Regular and Bold (consistent with existing views)
- Nag panel title uses `.font(.title2.bold())` — matches RESEARCH.md sketch exactly
- Settings section uses `.font(.footnote)` for both hint text and inline error messages — do NOT use `.font(.caption)` (5th size, exceeds 4-size maximum)
- No custom font sizes are declared; only system semantic sizes

**Source:** RESEARCH.md (LicenseNagView structure sketch); AccessibilitySettingsSection.swift (`.font(.body)` default)

## Color

This project uses only system semantic colors. No hex values are declared. All colors are adaptive (correct in both light and dark mode automatically).

| Role | SwiftUI Value | Usage |
|------|--------------|-------|
| Dominant (60%) | `.background` / `.regularMaterial` | Window background, form background |
| Secondary (30%) | `.secondary` foreground style | Hint text, footer labels, inactive states |
| Accent — licensed state | `.green` | "Licensed" status indicator (checkmark icon + label) |
| Accent — warning state | `.yellow` | Unlicensed warning indicator (triangle icon + label) |
| Destructive | `.red` (system default for destructive buttons) | "Deactivate License" button only |

Accent reserved for:
1. License status checkmark icon (`.green`) when activated — mirrors `GistSettingsSection` "Token saved" pattern
2. Warning triangle icon (`.yellow`) when not licensed — mirrors `AccessibilitySettingsSection` "Not Granted" pattern
3. Primary CTA "Buy a License" button in nag view uses `.borderedProminent` style — this renders in system accent color (blue by default, user-configurable)

**Semantic note:** `.foregroundStyle(.green)` / `.foregroundStyle(.yellow)` are used for status icons, consistent with existing `GistSettingsSection` and `AccessibilitySettingsSection` patterns. The app never hard-codes hex values.

**Source:** GistSettingsSection.swift (`.foregroundStyle(.green)`); AccessibilitySettingsSection.swift (`.foregroundStyle(.yellow)`); RESEARCH.md (LicenseNagView sketch: `.buttonStyle(.borderedProminent)`)

## Component Inventory

### LicenseNagView (new)

Hosted in `LicenseNagController` (activating NSPanel, `.titled` style). All measurements from RESEARCH.md Pattern 2.

```
LicenseNagView — VStack(spacing: 16), .padding(24), .frame(width: 380)
├── SF Symbol: "scissors" — .font(.system(size: 48))
├── Title: Text("Enjoying Clipsmith?") — .font(.title2.bold())
├── Body: Text("Clipsmith is free for personal use. If you use it for work,
│            please consider supporting development.")
│         — .multilineTextAlignment(.center), .font(.body), .foregroundStyle(.secondary)
├── HStack(spacing: 8)
│   ├── Button "Sponsor on GitHub" — .buttonStyle(.bordered)
│   └── Button "Buy a License" — .buttonStyle(.borderedProminent)
└── Button "I Already Have a License" — .buttonStyle(.plain), .font(.footnote)
    — posts notification to open Settings > License tab, then dismisses panel
```

**Panel behavior:**
- `makeKeyAndOrderFront(nil)` + `NSApp.setActivationPolicy(.regular)` to show
- `windowWillClose` restores `.accessory` activation policy (same as SnippetWindowView pattern)
- No modal sheet; the nag is a standalone window the user can dismiss

### LicenseSettingsSection (new Settings tab)

Follows the `Form { Section {} }` pattern used by every existing settings section.

```
LicenseSettingsSection — Form
└── Section("License")
    ├── [UNLICENSED STATE]
    │   ├── TextField "License Key" — bound to @State var licenseKeyInput: String
    │   ├── Button "Activate License" — .disabled(licenseKeyInput.trimmingCharacters.isEmpty)
    │   │   calls licenseService.activate(key:)
    │   ├── ProgressView (inline) — visible when licenseService.isValidating == true
    │   ├── Text(licenseService.lastError) — .font(.footnote), .foregroundStyle(.red) — visible when error != nil
    │   └── Link "Buy a License →" — opens Lemon Squeezy purchase URL via NSWorkspace
    │       .font(.footnote), .foregroundStyle(.secondary)
    └── [LICENSED STATE]
        ├── HStack
        │   ├── Image(systemName: "checkmark.circle.fill") — .foregroundStyle(.green)
        │   └── Text("License active") — .foregroundStyle(.primary)
        ├── Text("Licensed to: {customerEmail}") — .font(.footnote), .foregroundStyle(.secondary)
        └── Button("Deactivate License") — .foregroundStyle(.red), .buttonStyle(.link)
            calls licenseService.deactivate()
```

**Tab entry in SettingsView:**
```swift
LicenseSettingsSection()
    .tabItem {
        Label("License", systemImage: "key")
    }
```

## Copywriting Contract

All copy is in American English. Tone: friendly, non-guilt-inducing, developer-appropriate.

| Element | Copy |
|---------|------|
| Nag panel title | "Enjoying Clipsmith?" |
| Nag panel body | "Clipsmith is free for personal use. If you use it for work, please consider supporting development." |
| Primary CTA (nag) | "Buy a License" |
| Secondary CTA (nag) | "Sponsor on GitHub" |
| Tertiary link (nag) | "I Already Have a License" |
| Settings tab label | "License" |
| Settings section heading | "License" |
| TextField placeholder | "License Key" |
| Activate button label | "Activate License" |
| Licensed status label | "License active" |
| Licensed sub-label | "Licensed to: {customerEmail}" |
| Deactivate button label | "Deactivate License" |
| Purchase link label | "Buy a License →" |
| Purchase link hint | "One-time purchase — supports indie development." |
| Validating state | (ProgressView inline, no label) |
| Empty license key error | "Enter a license key to activate." |
| Invalid key error | "License key not found. Check your key and try again." |

## Interaction Contract

### LicenseNagController

| Trigger | Behavior |
|---------|----------|
| App launch, unlicensed, 30-day gate passes | Show nag panel after 1.0s delay; `NSApp.setActivationPolicy(.regular)` |
| "Buy a License" clicked | `NSWorkspace.shared.open(purchaseURL)` — opens in browser; panel stays open |
| "Sponsor on GitHub" clicked | `NSWorkspace.shared.open(sponsorURL)` — opens in browser; panel stays open |
| "I Already Have a License" clicked | Post notification → Settings opens to License tab; panel closes |
| Window close button (red dot) | Panel closes; `windowWillClose` restores `.accessory` activation policy |
| App already licensed on launch | Nag panel never shown (gated by `licenseService.isLicensed`) |

### LicenseSettingsSection

| Trigger | Behavior |
|---------|----------|
| User types in TextField | `licenseKeyInput` updates; Activate button enabled when non-empty after trimming |
| "Activate License" tapped | Button disabled; ProgressView shown; `licenseService.activate(key:)` called async |
| Activation success | Licensed state shown; ProgressView hidden; `lastError` cleared |
| Activation failure | Error message shown below TextField in `.red .footnote`; ProgressView hidden |
| "Deactivate License" tapped | Calls `licenseService.deactivate()` immediately (no confirmation); returns to unlicensed state |
| Settings opened via "I Already Have a License" | Tab selection navigates to License tab (via `NotificationCenter` → `SettingsView` tab switch) |

### Focus and keyboard behavior

- TextField in LicenseSettingsSection receives first responder automatically when the tab is selected (standard SwiftUI `Form` behavior)
- Return/Enter key in the license key TextField triggers "Activate License" (`.onSubmit` binding)
- No keyboard shortcuts are added to the nag panel — it is informational, not a workflow step
