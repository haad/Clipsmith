<objective>
Build the bezel HUD: a non-activating NSPanel hosting a SwiftUI view with keyboard navigation and search filtering.

Purpose: This is the primary interaction surface for Flycut -- the floating overlay that lets users browse and select clippings without leaving their current app. The bezel must appear without stealing focus, respond to keyboard navigation, filter by search text, and dismiss on paste/Escape/click-outside.

Output: BezelViewModel (pure-Swift testable logic), BezelController (NSPanel subclass), BezelView (SwiftUI content), and unit tests for both the view model and controller configuration.
</objective>

  <behavior>
    - navigateDown increments selectedIndex when not at last item; no-op at last
    - navigateUp decrements selectedIndex when not at first item; no-op at first
    - navigateToFirst sets selectedIndex to 0
    - navigateToLast sets selectedIndex to filteredClippings.count - 1 (or 0 if empty)
    - navigateUpTen decrements by 10 but clamps at 0
    - navigateDownTen increments by 10 but clamps at last index
    - filteredClippings returns all items when searchText is empty
    - filteredClippings returns only items containing searchText (case-insensitive) when non-empty
    - Setting searchText resets selectedIndex to 0
    - currentClipping returns nil when clippings array is empty
    - currentClipping returns the item at selectedIndex from filteredClippings
  </behavior>

Create `FlycutSwift/Views/BezelController.swift` as `@MainActor final class BezelController: NSPanel`:

1. **init()**: Call `super.init(contentRect: NSRect(x:0,y:0,width:400,height:280), styleMask: [.nonactivatingPanel, .borderless, .fullSizeContentView], backing: .buffered, defer: false)`. CRITICAL: `.nonactivatingPanel` MUST be in the init styleMask -- setting it afterward silently fails (WindowServer tag not updated).

2. **Window properties**:
   - `level = NSWindow.Level(rawValue: Int(CGWindowLevelForKey(.screenSaverWindow)) + 1)` -- above fullscreen apps
   - `collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary, .transient]` -- all Spaces + fullscreen + Stage Manager compatible
   - `isOpaque = false`, `backgroundColor = .clear`, `hasShadow = true`
   - `isMovableByWindowBackground = false`, `isReleasedWhenClosed = false` (reuse across show/hide)
   - `canBecomeKey` override returns `true` (text input works in non-activating panel)
   - `canBecomeMain` override returns `false`

3. **Content view**: Create `BezelView()`, wrap in `NSHostingView`. Inject environment: `.environment(pasteService).environment(appTracker).modelContainer(FlycutApp.sharedModelContainer)`. Store the `BezelView` reference (or its view model) so `keyDown` can call navigation. The simplest approach: store a reference to the `BezelViewModel` instance that is shared between the controller and the view.

4. **show()**: Center on screen containing `NSEvent.mouseLocation` using `NSScreen.screens.first(where: { $0.frame.contains(NSEvent.mouseLocation) }) ?? NSScreen.main ?? NSScreen.screens[0]`. Set frame origin to center. Call `makeKeyAndOrderFront(nil)`. Do NOT call `NSApp.activate()`. Register global click-outside monitor via `NSEvent.addGlobalMonitorForEvents(matching: .leftMouseDown)` that checks `!frame.contains(NSEvent.mouseLocation)` and calls `hide()`.

5. **showWithSearch()**: Same as `show()` but sets `viewModel.searchText = ""` and focuses the search field.

6. **hide()**: Call `orderOut(nil)`. Remove global monitor via `NSEvent.removeMonitor(globalMonitor); globalMonitor = nil`. Reset `viewModel.selectedIndex = 0` and `viewModel.searchText = ""`.

7. **keyDown(with:)**: Switch on `event.keyCode`:
   - 53 (Escape): `hide()`
   - 36, 76 (Return, Enter): Call paste and hide -- `Task { @MainActor in await pasteAndHide() }`
   - 125, 124 (Down, Right): `viewModel.navigateDown()`
   - 126, 123 (Up, Left): `viewModel.navigateUp()`
   - 121 (Page Down): `viewModel.navigateDownTen()`
   - 116 (Page Up): `viewModel.navigateUpTen()`
   - 119 (End): `viewModel.navigateToLast()`
   - 115 (Home): `viewModel.navigateToFirst()`
   - default: `super.keyDown(with: event)` (passes printable chars to SwiftUI TextField)

8. **pasteAndHide()**: Get `viewModel.currentClipping`, call `pasteService.paste(content:into:appTracker.previousApp)`, then `hide()`.

Create `FlycutSwift/Views/BezelView.swift` as a SwiftUI struct:
- `@Query(sort: \FlycutSchemaV1.Clipping.timestamp, order: .reverse) private var clippings`

- `@AppStorage(AppSettingsKeys.displayNum) private var displayNum: Int = 10`
- In `.onChange(of: clippings)` and `.onAppear`, map clippings to `viewModel.clippings = clippings.prefix(displayNum).map(\.content)`
- Body layout: VStack with search TextField at top bound to `$viewModel.searchText`, ScrollView with `Text(viewModel.currentClipping ?? "")` in center, navigation label `Text(viewModel.navigationLabel)` at bottom
- Background: `.ultraThinMaterial` with `RoundedRectangle(cornerRadius: 16)` clip shape
- Empty states: "No clippings" when clippings empty, "No matches" when filteredClippings empty
