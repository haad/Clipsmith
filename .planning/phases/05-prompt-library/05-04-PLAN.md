  <files>FlycutSwift/Views/Prompts/PromptLibraryView.swift</files>
  <action>
    Create `FlycutSwift/Views/Prompts/PromptLibraryView.swift`:

    A master-detail prompt browser mirroring SnippetListView patterns.

    **Environment / Query / State:**
    ```swift
    @Environment(PasteService.self) private var pasteService
    @Environment(AppTracker.self) private var appTracker
    @Environment(\.modelContext) private var modelContext
    @Query(sort: \FlycutSchemaV2.PromptLibraryItem.title)
    private var allPrompts: [FlycutSchemaV2.PromptLibraryItem]
    @State private var searchText: String = ""
    @State private var selectedPromptID: PersistentIdentifier?
    @State private var promptStore: PromptLibraryStore?
    @State private var snippetStore: SnippetStore?
    @FocusState private var isSearchFocused: Bool
    ```

    **Filtered prompts (computed):**
    Support `#category` search syntax per CONTEXT.md:
    - If searchText starts with "#", extract category name (up to first space), filter by category, search remaining text
    - For "#My Prompts" or "#my", filter isUserCreated==true
    - Otherwise, search title and content with localizedCaseInsensitiveContains
    - Also check category field for non-# searches so searching "coding" shows coding prompts

    **Body (VStack):**
    1. **Search bar** (matches SnippetListView): magnifying glass icon + TextField + clear button

    2. **HSplitView:**

       **Left panel (min 200, ideal 240, max 300):**
       - Toolbar row: "+" button (cmd+N) to create new user prompt
       - List of filteredPrompts with selection binding:
         Each row shows:
         - Icon: `book.fill` for library prompts (`.secondary`), `person.fill` for user-created (`.accentColor`)
         - Title text (lineLimit 1)
         - Category badge: small pill with category name (colored: coding=blue, writing=green, analysis=purple, creative=orange, My Prompts=gray)
         - If isUserCustomized: small orange "edited" text or dot
         - Double-click pastes prompt (pastePrompt())
         - Context menu: Delete (for user-created), Revert to Original (for customized library prompts)
       - `.onKeyPress(.return)` pastes selected prompt
       - Empty state: "No prompts" or "No results"

       **Right panel (detail view):**
       If a prompt is selected:
       - **Header:** Title (editable TextField), category badge
       - **Content area:** TextEditor for prompt content (editable)
         - For library prompts, editing triggers isUserCustomized=true on save
         - {{variable}} tokens should be visually indicated — use a note below the editor listing detected variables (via TemplateSubstitutor.extractVariables)
       - **Variables note:** "Variables: {{clipboard}}, {{name}}" — shows detected variables as a `.caption .secondary` text below the editor
       - **Action buttons row:**
         - "Save to My Snippets" button — calls snippetStore.insert() with prompt title/content, tags ["prompt", category]
         - "Revert to Original" button (only if isUserCustomized && !isUserCreated) — calls promptStore.revertToOriginal(), shows confirmation via NSAlert first
       - **Auto-save on edit:** Use onChange of the edited title/content to debounce and call promptStore.update() after a short delay (or on blur/selection change)
       - If no prompt selected: "Select a prompt" placeholder

    **createNewPrompt():**
    - Insert via promptStore.insert(id: UUID, title: "New Prompt", content: "", category: "My Prompts", version: 1, isUserCreated: true)
    - Select the new prompt

    **pastePrompt():**
    - Read clipboard, build variables dict (clipboard + user-defined from UserDefaults), substitute, paste via PasteService, close window
    - Same pattern as SnippetListView.pasteSnippet() — close window, set accessory policy, paste

    **setupStores():**
    - Initialize promptStore and snippetStore from modelContext.container (same pattern as SnippetListView.setupSnippetStore)

    **Keyboard shortcuts (hidden buttons):**
    - cmd+F focuses search
    - cmd+Delete deletes selected prompt (user-created only)

    Add to Xcode project file (FlycutSwift target). Create the `Views/Prompts/` directory group in the project if needed.
