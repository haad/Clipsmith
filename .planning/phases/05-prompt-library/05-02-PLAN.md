  <name>Task 1: PromptSyncService — HTTP fetch + bundled JSON loading</name>
  <files>FlycutSwift/Services/PromptSyncService.swift</files>
  <action>
    Create `FlycutSwift/Services/PromptSyncService.swift`:

    1. Define error enum:
       ```swift
       enum PromptSyncError: Error, LocalizedError {
           case invalidURL
           case httpError(Int)
           case networkError(Error)
           case decodingError(Error)
           case bundleNotFound
           // Each case has errorDescription
       }
       ```

    2. Create `@MainActor @Observable final class PromptSyncService`:
       - Properties: `private let session: URLSession`, `var isSyncing: Bool = false`, `var lastError: String? = nil`
       - `init(session: URLSession = .shared)`

       - `func loadBundledPrompts(store: PromptLibraryStore) async throws`:
         - Finds "prompts" resource of type "json" in Bundle.main
         - Decodes as PromptCatalog
         - For each PromptDTO, calls `store.upsert(remote:)`
         - This is the offline first-launch path (PMPT-01)

       - `func syncFromURL(_ urlString: String, store: PromptLibraryStore) async throws`:
         - Set isSyncing=true, lastError=nil
         - Validate URL (throw .invalidURL if bad)
         - Create URLRequest with the URL
         - Call session.data(for:), wrap errors in .networkError
         - Check HTTPURLResponse status (200 OK, else throw .httpError)
         - Decode response data as PromptCatalog, wrap errors in .decodingError
         - For each prompt in catalog.prompts, call `store.upsert(remote:)`
         - Update UserDefaults promptLibraryLastSync with Date.now (ISO 8601 string)
         - Set isSyncing=false
         - On error: set lastError = error.localizedDescription, set isSyncing=false, rethrow

       IMPORTANT per CONTEXT.md: NO SwiftGitX, NO git clone/pull. Just a simple URLSession HTTP GET of a single prompts.json file. Same pattern as GistService network calls.

       The service is @MainActor @Observable so SwiftUI settings can observe isSyncing state directly. The actual network call is async and does not block the main thread.
