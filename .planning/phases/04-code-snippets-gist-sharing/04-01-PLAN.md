    **Schema update:**

    In `FlycutSchemaV1.swift`, add `var tags: [String] = []` to the Snippet @Model. Keep the existing `category` field for backward compatibility (existing data won't break). Update the `init()` to accept an optional `tags: [String] = []` parameter.

    **GREEN phase — implement SnippetStore:**

    Create `FlycutSwift/Services/SnippetStore.swift` as a `@ModelActor actor` mirroring ClipboardStore:

    ```swift
    @ModelActor
    actor SnippetStore {
        func insert(name: String, content: String, language: String?, tags: [String]) throws
        func fetchAll() throws -> [PersistentIdentifier]
        func fetchByLanguage(_ language: String) throws -> [PersistentIdentifier]
        func search(query: String) throws -> [PersistentIdentifier]
        func update(id: PersistentIdentifier, name: String, content: String, language: String?, tags: [String]) throws
        func delete(id: PersistentIdentifier) throws
        func content(for id: PersistentIdentifier) -> String?
        func snippet(for id: PersistentIdentifier) -> SnippetInfo?
    }
    ```

    Define `SnippetInfo` as a Sendable struct (mirrors ClippingInfo pattern):
    ```swift
    struct SnippetInfo: Sendable, Identifiable {
        let id: PersistentIdentifier
        let name: String
        let content: String
        let language: String?
        let tags: [String]
        let createdAt: Date
        let updatedAt: Date
    }
    ```

    Search implementation: use `#Predicate` with `localizedStandardContains` across name and content fields. For tags, since SwiftData `#Predicate` does not support `.contains()` on `[String]`, fetch all and filter in-memory for tag matches. This is acceptable for snippet collections (typically < 1000 items).

    PITFALL: `(snippet.category ?? "").localizedStandardContains(searchText)` pattern for optional fields (from RESEARCH.md Pitfall 2).

    fetchAll sorts by `updatedAt` descending (most recently edited first — natural for a snippet editor).

    Always call `modelContext.save()` immediately after mutations (PITFALL: temporary PersistentIdentifier before save — from RESEARCH.md Pitfall 3).

    Use `OSLog.Logger` with category "SnippetStore" for structured logging.
