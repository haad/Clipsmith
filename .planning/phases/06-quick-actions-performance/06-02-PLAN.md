    1. Guard viewModel.currentClipping is not nil
    2. Apply transform to get result string
    3. Write result to NSPasteboard.general (clearContents + setString forType:.string)
    4. Set clipboardMonitor?.blockedChangeCount = NSPasteboard.general.changeCount (RESEARCH.md Pitfall 2: prevent self-capture)
    5. Insert transformed content into ClipboardStore as new entry with sourceAppName "Flycut (transformed)"
    6. Do NOT auto-paste -- the user presses Enter after reviewing (RESEARCH.md Open Question 1)

    `actionCopyAsRTF` implementation:
    1. Guard viewModel.currentClipping
    2. Call TextTransformer.copyAsRTF(content) to get RTF Data
    3. Write to NSPasteboard.general with .rtf type
    4. Set blockedChangeCount to prevent self-capture
    5. Hide the bezel (RTF is on pasteboard, user can Cmd-V)

    `actionShareAsGist` implementation:
    1. Guard viewModel.currentClipping
    2. Post .flycutShareAsGist notification with userInfo["content"] (reuses existing AppDelegate handler -- zero new Gist code needed)
    3. Hide the bezel

    Also add a keyboard shortcut for quick actions: in keyDown, handle Tab key (keyCode 48) to show the quick action menu programmatically via `showQuickActionMenu(at:)`. This gives keyboard-only users access to quick actions without right-clicking.

    **STEP 1: Write tests FIRST (TDD Red phase)**

    ```swift
    // MARK: - Adaptive polling tests (PERF-02)

    /// After start(), activityMonitor must be non-nil (NSEvent global monitor registered).
    func testActivityMonitorRegistered() throws {
        let monitor = ClipboardMonitor()
        monitor.start()
        // activityMonitor is private — expose via an internal computed property
        // `var hasActivityMonitor: Bool` that returns activityMonitor != nil
        XCTAssertTrue(monitor.hasActivityMonitor,
                      "activityMonitor should be registered after start()")
        monitor.stop()
        XCTAssertFalse(monitor.hasActivityMonitor,
                       "activityMonitor should be nil after stop()")
    }

    /// When the timer interval matches the expected interval (delta < 0.01),
    /// scheduleTimer should NOT be called again (avoid unnecessary timer churn).
    func testTimerNotRecreatedWhenIntervalUnchanged() throws {
        let monitor = ClipboardMonitor()
        monitor.start()
        // After start(), timer is set to activeInterval. Simulate an "active" tick
        // by calling checkPasteboardAdaptive() — the timer should NOT be recreated
        // because the interval is already correct.
        // Expose `var currentTimerInterval: TimeInterval?` (returns timer?.timeInterval)
        // and `var timerRecreationCount: Int` (incremented in scheduleTimer, test-only).
        let initialCount = monitor.timerRecreationCount
        monitor.checkPasteboardAdaptive()
        XCTAssertEqual(monitor.timerRecreationCount, initialCount,
                       "Timer should not be recreated when interval is unchanged")
        monitor.stop()
    }
    ```

    To make these tests compilable, add two internal test-support properties to ClipboardMonitor:
    - `var hasActivityMonitor: Bool { activityMonitor != nil }` — exposes whether the NSEvent monitor is registered
    - `private(set) var timerRecreationCount: Int = 0` — incremented at the top of `scheduleTimer(interval:)` so tests can detect unnecessary recreation
    - Make `checkPasteboardAdaptive()` internal (not private) so tests can call it directly (same pattern as existing `checkPasteboard()`)

    Run tests -- they must FAIL because `scheduleTimer`, `checkPasteboardAdaptive`, `hasActivityMonitor`, and `timerRecreationCount` do not exist yet.

    **STEP 2: Implement adaptive polling (TDD Green phase)**

    Modify ClipboardMonitor to implement adaptive polling (PERF-02). Add these private properties:

    ```
    private var activityMonitor: Any?
    private var lastActivityDate: Date = .now
    private let activeInterval: TimeInterval = 0.5
    private let idleInterval: TimeInterval = 3.0
    private let idleThreshold: TimeInterval = 30.0
    private(set) var timerRecreationCount: Int = 0
    ```

    Add `var hasActivityMonitor: Bool { activityMonitor != nil }` internal computed property.

    Modify `start()`:
    1. Keep existing guard and lastChangeCount setup
    2. Read clipboardPollingInterval from UserDefaults -- use it as activeInterval override if > 0 (respect user setting)
    3. Call `scheduleTimer(interval: activeInterval)` instead of creating timer directly
    4. Call `registerActivityMonitor()` to start tracking user activity
    5. Set isMonitoring = true

    Add `scheduleTimer(interval:)` private method:
    1. Increment timerRecreationCount
    2. Invalidate existing timer
    3. Create new Timer with the given interval, repeating, calling checkPasteboardAdaptive()
    4. Add to RunLoop.current with .common mode (CRITICAL: maintains existing pattern)

    Add `checkPasteboardAdaptive()` method (internal, not private -- same pattern as checkPasteboard):
    1. Calculate time since lastActivityDate
    2. Determine if idle (sinceActivity > idleThreshold)
    3. Compute expectedInterval = idle ? idleInterval : activeInterval
    4. Compare with current timer?.timeInterval -- only call scheduleTimer if abs(difference) > 0.01 (RESEARCH.md Pitfall 5)
    5. Call existing checkPasteboard()

    Add `registerActivityMonitor()` private method:
    1. Remove existing if any
    2. Use NSEvent.addGlobalMonitorForEvents(matching: [.mouseMoved, .keyDown, .leftMouseDown, .rightMouseDown, .scrollWheel])
    3. In handler: Task { @MainActor [weak self] in self?.lastActivityDate = .now } (Swift 6 Sendable-compliant pattern, already used in codebase for flagsMonitor)

    Modify `stop()`:
    1. Keep existing timer?.invalidate() and isMonitoring = false
    2. Add: remove activity monitor (if let m = activityMonitor { NSEvent.removeMonitor(m); activityMonitor = nil })

    The existing `checkPasteboard()` method is unchanged -- `checkPasteboardAdaptive()` calls it after the interval adaptation logic.

    Run tests -- they must PASS.
