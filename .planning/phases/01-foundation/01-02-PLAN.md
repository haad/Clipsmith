<task type="auto">
  <name>Task 1: Add KeyboardShortcuts SPM dependency, create AppSettings and settings keys</name>
  <files>
    FlycutSwift/Settings/AppSettings.swift,
    FlycutSwift/Settings/AppSettingsKeys.swift,
    FlycutSwift/Settings/KeyboardShortcutNames.swift,
    FlycutSwift.xcodeproj/project.pbxproj
  </files>
  <action>
    **Add KeyboardShortcuts SPM package:**
    - URL: `https://github.com/sindresorhus/KeyboardShortcuts`
    - Version requirement: 2.4.0 or newer (Swift 6 confirmed per research)
    - Add via `swift package` CLI or by editing Package.swift / .xcodeproj
    - Link `KeyboardShortcuts` framework to the FlycutSwift target

    **AppSettingsKeys.swift:**
    - `enum AppSettingsKeys` (no cases — namespace only)
    - String constants: `rememberNum`, `displayNum`, `displayLen`, `plainTextPaste`, `pasteSound`, `launchAtLogin`, `suppressAccessibilityAlert`

    **AppSettings.swift:**
    - `@Observable final class AppSettings`
    - Each property uses `@ObservationIgnored @AppStorage(AppSettingsKeys.xxx)` — CRITICAL: both annotations required (Pitfall 3 from research: @Observable + @AppStorage compile error without @ObservationIgnored)
    - Properties with defaults:
      - `rememberNum: Int = 99` (max history size)
      - `displayNum: Int = 10` (clippings shown in menu)
      - `displayLen: Int = 40` (characters per preview)
      - `plainTextPaste: Bool = false`
      - `pasteSound: Bool = false`
      - `launchAtLogin: Bool = false`
      - `suppressAccessibilityAlert: Bool = false`

    **KeyboardShortcutNames.swift:**
    - `import KeyboardShortcuts`
    - `extension KeyboardShortcuts.Name` with:
      - `static let activateBezel = Self("activateBezel")`
      - `static let activateSearch = Self("activateSearch")`
    - These are name stubs only — actual hotkey registration happens in Phase 2

    Build must succeed with strict concurrency.
  </action>
  <verify>
    <automated>xcodebuild build -project FlycutSwift.xcodeproj -scheme FlycutSwift -destination "platform=macOS" 2>&1 | tail -5</automated>
  </verify>
  <done>KeyboardShortcuts SPM package resolves and links. AppSettings compiles with @ObservationIgnored @AppStorage pattern. KeyboardShortcuts.Name stubs for activateBezel and activateSearch exist. Build succeeds.</done>
</task>

<task type="auto">
  <name>Task 2: Build SettingsView with General and Shortcuts tabs, wire SMAppService and AccessibilityMonitor</name>
  <files>
    FlycutSwift/Views/Settings/SettingsView.swift,

  <action>
    **SettingsView.swift:**
    - `TabView` with two tabs: General (gearshape icon) and Shortcuts (keyboard icon)
    - `.frame(minWidth: 420, minHeight: 300)`
    - Follow Pattern 7 from research exactly

    **GeneralSettingsTab.swift:**
    - **History section:** `Stepper` for `rememberNum` (range 1...999, label "Remember up to N clippings"), `Stepper` for `displayNum` (range 1...99, label "Show N in menu"), `Stepper` for `displayLen` (range 10...200, label "Preview length")
    - **Paste behavior section:** `Toggle` for `plainTextPaste` ("Always paste as plain text"), `Toggle` for `pasteSound` ("Play sound on paste")
    - **Launch section:** `Toggle` for `launchAtLogin` ("Launch Flycut at login")
      - On `onChange(of: launchAtLogin)`: call `setLaunchAtLogin(enabled)` which uses `SMAppService.mainApp.register()` / `.unregister()` — always check `.status` before calling (Pitfall 4 from research: status mismatch)
      - On `.onAppear`: sync `launchAtLogin` with `SMAppService.mainApp.status == .enabled` to reflect system ground truth
    - **Accessibility section:** Use `AccessibilitySettingsSection` (below)
    - Bind to AppSettings via `@Environment` or `@State` — read from `@AppStorage` directly in the view if simpler

    **AccessibilitySettingsSection.swift:**
    - Read `AccessibilityMonitor.isTrusted` from environment
    - Show green checkmark + "Accessibility: Granted" if trusted
    - Show yellow warning + "Accessibility: Not Granted" with "Open System Settings" button if not trusted
    - The button calls `accessibilityMonitor.openAccessibilitySettings()` — opens System Settings to Accessibility pane
    - NEVER call `AXIsProcessTrustedWithOptions(prompt: true)` from this view

    **HotkeySettingsTab.swift:**
    - `import KeyboardShortcuts`
    - `Form` with two labeled rows:
      - "Activate Clipboard" → `KeyboardShortcuts.Recorder(for: .activateBezel)`
      - "Activate Search" → `KeyboardShortcuts.Recorder(for: .activateSearch)`
    - Note in a footer: "Hotkey behavior will be active after restart" (actual registration is Phase 2)

    **Wire into FlycutApp.swift:**
    - Replace placeholder SettingsView with the real `SettingsView()`
    - Inject `AppSettings` and `AccessibilityMonitor` into environment so all settings views can access them
    - Ensure `Settings { SettingsView().environment(...) }` passes through needed objects

    Build must succeed. Use `import ServiceManagement` in GeneralSettingsTab. Use `import OSLog` for any error logging on SMAppService failures.
