  <action>
    Create a new Xcode project for macOS App target named "FlycutSwift" using Swift and SwiftUI. Minimum deployment: macOS 15.0.

    **Info.plist configuration:**
    - Set `LSUIElement = YES` (Application is agent)
    - Bundle identifier: `com.generalarcade.flycut` (preserve existing for accessibility trust continuity)
    - No `NSMainNibFile` entry (SwiftUI app, no NIBs)

    **Entitlements:**
    - Disable App Sandbox (`com.apple.security.app-sandbox = false`)
    - Enable Hardened Runtime
    - Add `com.apple.security.automation.apple-events` entitlement

    **AppDelegate.swift:**
    - `NSObject, NSApplicationDelegate` class
    - In `applicationDidFinishLaunching`: call `NSApp.setActivationPolicy(.accessory)` — BOTH this AND LSUIElement are required per research (Pitfall 1: activation policy split-brain)
    - Register UserDefaults defaults: `rememberNum: 99`, `displayNum: 10`, `displayLen: 40`, `plainTextPaste: false`, `pasteSound: false`, `launchAtLogin: false`, `suppressAccessibilityAlert: false`

    **FlycutApp.swift (@main):**
    - Use `@NSApplicationDelegateAdaptor(AppDelegate.self)`
    - Body contains `MenuBarExtra("Flycut", systemImage: "doc.on.clipboard")` with `.menuBarExtraStyle(.menu)` — NOT `.window` (Pitfall 4)
    - Body contains `Settings { SettingsView() }` (placeholder SettingsView with Text("Settings coming soon"))
    - Attach `.modelContainer(sharedModelContainer)` to MenuBarExtra
    - Create `sharedModelContainer` as a static property:
      - Schema from `FlycutSchemaV1.models`
      - URL at `~/Library/Application Support/Flycut/clipboard.sqlite`
      - Create intermediate directories
      - Use `FlycutMigrationPlan.self`

    **FlycutSchemaV1.swift (VersionedSchema):**
    - `enum FlycutSchemaV1: VersionedSchema` with `versionIdentifier = Schema.Version(1, 0, 0)`
    - Contains `Clipping` @Model: `content: String`, `type: String`, `sourceAppName: String?`, `sourceAppBundleURL: String?`, `timestamp: Date`, `isFavorite: Bool`, `displayOrder: Int` — all with default values
    - Contains `Snippet` @Model: `name: String`, `content: String`, `language: String?`, `category: String?`, `createdAt: Date`, `updatedAt: Date`
    - Contains `GistRecord` @Model: `gistID: String`, `gistURL: String`, `filename: String`, `createdAt: Date`
    - Follow exact patterns from 01-RESEARCH.md Pattern 2

    **FlycutMigrationPlan.swift:**
    - `enum FlycutMigrationPlan: SchemaMigrationPlan`
    - `schemas: [FlycutSchemaV1.self]`
    - `stages: []` (empty — first version, no migration needed)

    **MenuBarView.swift (placeholder):**
    - Simple SwiftUI View with `Text("No clippings yet")` and a `Divider()` followed by `Button("Quit") { NSApplication.shared.terminate(nil) }`

    **Build settings:**
    - `SWIFT_STRICT_CONCURRENCY = complete`
    - `ENABLE_TESTABILITY = YES`
    - `ALWAYS_SEARCH_USER_PATHS = NO`

    Use `os.Logger` for any diagnostic output — never `print()`.
  </action>

  <name>Task 2: Add AccessibilityMonitor service and wire to app</name>
  <files>
    FlycutSwift/Services/AccessibilityMonitor.swift,

    FlycutSwift/App/AppDelegate.swift
  </files>
  <action>
    **AccessibilityMonitor.swift:**
    - `@Observable @MainActor final class AccessibilityMonitor`
    - `private(set) var isTrusted: Bool = false`
    - `private var timer: Timer?`
    - `func start()`: set initial `isTrusted = AXIsProcessTrusted()`, then create `Timer.scheduledTimer(withTimeInterval: 5.0, repeats: true)` that updates `isTrusted` on `@MainActor`
    - `func stop()`: invalidate and nil the timer
    - `func openAccessibilitySettings()`: open `x-apple.systempreferences:com.apple.settings.PrivacySecurity.extension?Privacy_Accessibility` via `NSWorkspace.shared.open()`
    - Use `import ApplicationServices` for `AXIsProcessTrusted()`
    - Use `os.Logger` with subsystem from Bundle.main.bundleIdentifier and category "AccessibilityMonitor"
    - CRITICAL: Never call `AXIsProcessTrustedWithOptions(prompt: true)` in the timer — only on explicit user action (Pitfall 5 from research)

    **Wire into AppDelegate:**
    - Create `AccessibilityMonitor` instance as a property on AppDelegate
    - Call `accessibilityMonitor.start()` in `applicationDidFinishLaunching`
    - Make the monitor accessible to SwiftUI views via `@Environment` or by injecting into the environment

    **Wire into FlycutApp.swift:**
    - Pass `accessibilityMonitor` into the SwiftUI environment so Settings and MenuBar views can observe it

    The project must still build cleanly with strict concurrency.
  </action>

    <automated>xcodebuild build -project FlycutSwift.xcodeproj -scheme FlycutSwift -destination "platform=macOS" 2>&1 | tail -5</automated>
  </verify>
