    Private static stored properties:
    - `safeMathRegex: NSRegularExpression` compiled with `try!` from raw-string pattern `#"^[\d\s()\.\+\-\*/,eE]+$"#`. (The `*` is already in the class. `**` is `* *` adjacent — both are in the class. Do NOT add `^` to the safe set; preprocessing strips it first. Do NOT add `sin`/`cos` letters to the safe set; the constant-arg pre-processor strips them before the gate runs.)
    - `funcNames: [String]` = `["sqrt", "abs", "ceiling", "floor", "ln", "log", "exp"]` (sin/cos NOT included — they are handled by the constant-arg pre-processor, not stripped from the safe-chars gate input)
    - `sinCosRegex: NSRegularExpression` compiled with `try!` from raw-string pattern `#"\b(sin|cos)\(\s*(-?[0-9]+(?:\.[0-9]+)?)\s*\)"#`, options `[.caseInsensitive]`. This matches `sin(0)`, `cos(3.14)`, `sin(-0.5)`, `cos( 1.5 )` — a single bare numeric literal argument. Nested expressions like `sin(2+2)` or `sin(sqrt(4))` do NOT match (the inner `(` after `sin(` is not a digit/dot/minus).

    Public static methods:

    `@MainActor static func evaluate(_ rawText: String) -> Double?`
    Algorithm:
    1. Trim whitespace. If empty, return `nil`.
    2. Constant-arg sin/cos pre-processor (D-05 in-scope half):
       - Use `sinCosRegex.matches(in: expr, range: ...)` to find ALL matches in the expression.
       - Iterate matches in REVERSE order (highest match.range.location first) so substring replacement does not shift the indices of remaining matches.
       - For each match: extract the function name (group 1, lowercased) and the numeric literal (group 2). Parse the literal with `Double(_:)`. Compute `let val = (fn == "sin") ? Foundation.sin(arg) : Foundation.cos(arg)`. Format the result with `String(format: "%.15g", val)` and substitute the full match range with that literal string.
       - If the literal parse fails for any match, return `nil` (defensive — should not happen because the regex requires `\d+`).
       - After this step, `sin(0)` becomes `0` (as a literal), `cos(3.14)` becomes `-0.9999987317275395`, etc. The expression is now plain arithmetic and the safe-chars gate can accept it.
    3. Preprocess `^` → `**`: `replacingOccurrences(of: #"(?<![*])\^(?![*])"#, with: "**", options: .regularExpression)` — this avoids accidentally double-replacing if user already typed `**`.
    4. Build a `stripped` copy of the lowercased expression and remove each entry from `funcNames` (`.replacingOccurrences(of: fn, with: "")`). Also defensively strip any remaining `sin` / `cos` tokens (these only appear if the pre-processor didn't match — i.e., nested-expression form — and we want the safe-chars gate to reject them rather than crash NSExpression). The remaining text must then match `safeMathRegex`. If it does not match, return `nil` WITHOUT touching `NSExpression`. This is the T-12-01 mitigation gate.
    5. Construct `let nsExpr = NSExpression(format: expr)` and call `nsExpr.expressionValue(with: nil, context: nil)`. Cast to `NSNumber` — return `nil` on cast failure.
    6. Extract `doubleValue`. Reject `isNaN` and `isInfinite` (covers division-by-zero per Pitfall 8). Return the `Double` otherwise.

    `nonisolated static func formatResult(_ value: Double) -> String`
    Algorithm (per D-06 and Pattern 3):
    1. If `value == value.rounded(.toNearestOrEven) && !value.isInfinite && abs(value) < 1e15`, treat as integer:
       - Build a `NumberFormatter` with `numberStyle = .decimal`, `locale = Locale(identifier: "en_US")` (matches research — D-06 says locale-aware grouping; we keep `en_US` for predictability and to match REPL-verified output. If the executor prefers `Locale.current`, document it in the SUMMARY but `en_US` keeps tests deterministic.), `maximumFractionDigits = 0`. Return `formatter.string(from: NSNumber(value: Int64(value)))` falling back to `"\(Int64(value))"`.
    2. Otherwise return `String(format: "%.6g", value)`.

    `nonisolated static func copyableResult(_ value: Double) -> String`
    Return `String(format: "%.10g", value)`. No thousands grouping — this is what gets copied to the clipboard so it remains paste-safe in code editors and spreadsheets.

    Document at the top of the file in the doc comment: `// D-05 scope (per RESEARCH.md Open Question 1): sqrt/pow are supported natively by NSExpression's format string. sin/cos are supported ONLY for constant-numeric arguments — sin(0), cos(3.14), sin(-0.5) — via a regex pre-processor that substitutes the literal evaluated value before NSExpression sees the expression. Nested forms like sin(2+2) or sin(sqrt(4)) are deferred and return nil. This honors D-05 for the common case (typing a number into sin/cos) while keeping NSExpression's exception model safe.`
