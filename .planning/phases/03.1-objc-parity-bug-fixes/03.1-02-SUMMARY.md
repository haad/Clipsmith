# Phase 03.1 Plan 02: Hotkey Hold-Navigate-Release + Sticky Bezel Summary

**One-liner:** Global NSEvent flags monitor auto-pastes on modifier release with optional stickyBezel setting to keep bezel open until Enter/Escape.

## What Was Built

### Task 1: Hotkey hold-navigate-release with flags monitor (Bug #3)

Implemented the core ObjC Flycut UX where holding the hotkey modifier and pressing the shortcut again navigates through clippings, then releasing all modifiers auto-pastes the selected item.

**BezelController.swift changes:**
- Added `var isHotkeyHold = false` — tracks whether bezel was opened in hold mode
- Added `private var flagsMonitor: Any?` — global monitor for modifier key events
- Added `registerFlagsMonitor()` — registers `NSEvent.addGlobalMonitorForEvents(.flagsChanged)` to detect when all modifiers are released; triggers `pasteAndHide()` via Task { @MainActor }
- Added `removeFlagsMonitor()` — removes monitor and nils the reference
- Updated `show()` — calls `registerFlagsMonitor()` when `isHotkeyHold` is true
- Updated `hide()` — calls `removeFlagsMonitor()` and resets `isHotkeyHold = false` on all exit paths
- Changed `pasteAndHide()` from `private` to `internal` — needed by flags monitor closure and future double-click paste

**AppDelegate.swift changes:**
- Updated `.activateBezel` hotkey handler: on first press reads `UserDefaults.standard.bool(forKey: AppSettingsKeys.stickyBezel)` and sets `bezelController.isHotkeyHold = !stickyBezel`

**AppSettingsKeys.swift:**
- Added `static let stickyBezel = "stickyBezel"` (prerequisite for AppDelegate compile)

**Tests added:**
- `testIsHotkeyHoldDefaultsFalse` — verifies isHotkeyHold defaults to false
- `testHideResetsIsHotkeyHold` — sets isHotkeyHold=true, calls hide(), asserts false

### Task 2: Sticky bezel setting + UI toggle (Bug #4)

Added the `stickyBezel` user preference that disables auto-paste-on-release, keeping the bezel open until the user presses Enter or Escape.

**AppSettings.swift:**
- Added `@ObservationIgnored @AppStorage(AppSettingsKeys.stickyBezel) var stickyBezel: Bool = false`

**AppDelegate.swift:**
- Added `"stickyBezel": false` to `UserDefaults.standard.register(defaults:)` call

**GeneralSettingsTab.swift:**
- Added `@AppStorage(AppSettingsKeys.stickyBezel) private var stickyBezel: Bool = false`
- Added `Toggle("Sticky bezel (stay open until Enter/Escape)", isOn: $stickyBezel)` in Paste Behavior section

## Key Decisions

- `pasteAndHide()` changed from `private` to `internal` — required by the flags monitor closure (which is a non-member closure) and noted in plan for future Bug #13 double-click paste
- Flags monitor uses `NSEvent.addGlobalMonitorForEvents` approach, NOT `flagsChanged(with:)` NSPanel override — non-activating panels don't reliably receive flagsChanged events via override
- `stickyBezel` read directly from `UserDefaults.standard` in AppDelegate hotkey handler — AppDelegate has no AppSettings instance; reading via UserDefaults.standard with the constant key is the correct pattern already used for other settings
