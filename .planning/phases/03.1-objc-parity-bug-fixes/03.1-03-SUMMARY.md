## Decisions Made

- **ClippingInfo value type bridge:** Carries PersistentIdentifier for delete-by-ID and sourceAppName/sourceAppBundleURL for future Wave 4 display (Bug #14). Keeps ViewModel free of @Model objects which are not Sendable across actor boundaries.
- **makeClippingInfos() Option A:** Uses in-memory SwiftData container to get valid PersistentIdentifiers in ViewModel tests — straightforward, avoids artificial abstractions.
- **pasteMovesToTop reads UserDefaults.standard directly:** Consistent with stickyBezel pattern already established in AppDelegate; AppDelegate has no AppSettings instance.
- **Delete key dispatches async deleteCurrentClipping via Task:** NSPanel keyDown is synchronous; crossing to @ModelActor ClipboardStore.delete requires async Task.
