**ClipboardEntry struct captures source app name/URL at copy time; duplicate re-copies promote existing clipping to history top via timestamp update instead of silent drop**

- `ClipboardMonitor.onNewClipping` now delivers `ClipboardEntry` with source app metadata captured via `NSWorkspace.shared.frontmostApplication`, filtering Flycut self-capture

## Decisions Made

- **ClipboardEntry in ClipboardMonitor.swift:** Co-located with the service that creates it; no need for a separate types file at this stage
- **ClipboardStore test accessors:** Adding `sourceAppName(for:)`, `sourceAppBundleURL(for:)`, `timestamp(for:)` to the actor avoids Swift 6 `Sendable` errors from accessing `container.mainContext` (which is `@MainActor`-isolated) from non-`@MainActor` test methods
- **nil-defaulted params:** `sourceAppName: String? = nil, sourceAppBundleURL: String? = nil` in `insert()` keeps all existing call sites unchanged — clean backward compatibility with no overloads needed

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Swift 6 Sendable compliance: replaced mainContext access in tests with actor accessor methods**
- **Found during:** Task 1 GREEN (test compilation)
- **Issue:** `testInsertWithSourceApp` and `testDuplicateMovedToTop` used `container.mainContext` to verify metadata — this is `@MainActor`-isolated and generates a Swift 6 error ("Non-Sendable type 'ModelContext' cannot exit main actor-isolated context") in a non-`@MainActor` test class
- **Fix:** Added `sourceAppName(for:)`, `sourceAppBundleURL(for:)`, and `timestamp(for:)` methods to `ClipboardStore`; tests use these actor-isolated accessors instead of the main context
- **Files modified:** `FlycutSwift/Services/ClipboardStore.swift`, `FlycutTests/ClipboardStoreTests.swift`
- **Verification:** All tests compile and pass under Swift 6 strict concurrency
- **Committed in:** `07119d3` (Task 1+2 GREEN commit)
