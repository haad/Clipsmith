## Recommended Architecture

### Overview

**Pattern:** Observable Controller architecture with actor-based system services

The existing Flycut uses a three-tier MVC pattern: AppController → FlycutOperator → FlycutStore. The Swift rewrite translates this into:

- **SwiftUI Views** observe `@Observable` controllers
- **Controllers** (`@Observable @MainActor`) manage state and coordinate services
- **Services** (actors/classes) handle system integration (pasteboard, hotkeys, accessibility)
- **SwiftData Models** (`@Model`) persist data

```
┌─────────────────────────────────────────────────┐
│                   SwiftUI Views                  │
│  MenuBarExtra  │  BezelView  │  SettingsView    │
│  SnippetEditor │  GistSheet                     │
└──────────┬──────────────────────────────────────┘
           │ observes
┌──────────▼──────────────────────────────────────┐
│              ClipboardController                 │
│  @Observable @MainActor                          │
│  - clippings: [Clipping]                        │
│  - stackPosition: Int                            │
│  - isBezelVisible: Bool                          │
│  - searchQuery: String                           │
└──────────┬──────────────────────────────────────┘
           │ coordinates
┌──────────▼──────────────────────────────────────┐
│                   Services                       │
│  PasteboardMonitor  │  HotkeyManager            │
│  PasteService       │  GistService              │
│  AccessibilityChecker                            │
└──────────┬──────────────────────────────────────┘
           │ persists via
┌──────────▼──────────────────────────────────────┐
│               SwiftData Models                   │
│  @Model Clipping  │  @Model Snippet             │

└─────────────────────────────────────────────────┘
```

## Components

### 1. App Shell (`FlycutApp`)

**Purpose:** SwiftUI app entry point, configures MenuBarExtra and SwiftData container.

```swift
@main
struct FlycutApp: App {
    @NSApplicationDelegateAdaptor(AppDelegate.self) var appDelegate

    var body: some Scene {
        MenuBarExtra("Flycut", systemImage: "doc.on.clipboard") {
            MenuBarView()
        }
        .menuBarExtraStyle(.menu)

        Settings {
            SettingsView()
        }
    }
}
```

- `LSUIElement = YES` in Info.plist (no dock icon)
- `@NSApplicationDelegateAdaptor` for `applicationDidFinishLaunching` setup
- `NSApp.setActivationPolicy(.accessory)` to prevent dock appearance

### 2. ClipboardController (`@Observable @MainActor`)

**Purpose:** Central state coordinator. Replaces `FlycutOperator` + parts of `AppController`.

**Responsibilities:**
- Owns the current clipping list (in-memory for fast UI)
- Manages stack position for keyboard navigation
- Controls bezel visibility
- Coordinates between PasteboardMonitor, PasteService, and SwiftData

**Key pattern — replaces Obj-C delegate chain:**
```swift
@Observable @MainActor
class ClipboardController {
    var clippings: [Clipping] = []
    var stackPosition: Int = 0
    var isBezelVisible: Bool = false
    var searchQuery: String = ""

    private let monitor: PasteboardMonitor
    private let pasteService: PasteService
    private let modelContext: ModelContext

    func handleNewClipping(_ content: String, type: String, sourceApp: String?) {
        // Deduplicate, add to in-memory list, persist via SwiftData
    }

    func pasteSelected() {
        let clipping = clippings[stackPosition]
        pasteService.paste(clipping.content)
        isBezelVisible = false
    }
}
```

### 3. PasteboardMonitor (Actor)

**Purpose:** Polls `NSPasteboard.general.changeCount` and detects new clipboard content. Replaces `pollPB:` timer in AppController.

**Key design decisions:**
- All `NSPasteboard` reads on `@MainActor` (NSPasteboard is not thread-safe)
- `blockedChangeCount` property prevents re-capturing Flycut's own paste operations (replaces `pbBlockCount`)
- Pasteboard type denylist for password exclusion

```swift
@MainActor
class PasteboardMonitor {
    private var lastChangeCount: Int = 0
    private var blockedChangeCount: Int? = nil
    private var timer: Timer?

    private let denylist: Set<String> = [
        "PasswordPboardType",
        "org.nspasteboard.TransientType",
        "org.nspasteboard.ConcealedType",
        "org.nspasteboard.AutoGeneratedType",
        "com.agilebits.onepassword"
    ]

    func blockNextChange() {
        blockedChangeCount = NSPasteboard.general.changeCount + 1
    }

    func checkPasteboard() -> (String, String)? {
        let current = NSPasteboard.general.changeCount
        guard current != lastChangeCount else { return nil }
        lastChangeCount = current

        if let blocked = blockedChangeCount, current == blocked {
            blockedChangeCount = nil
            return nil
        }

        // Check denylist, read content
        guard let types = NSPasteboard.general.types else { return nil }
        if types.contains(where: { denylist.contains($0.rawValue) }) { return nil }

        guard let string = NSPasteboard.general.string(forType: .string) else { return nil }
        guard !string.isEmpty else { return nil }
        return (string, NSPasteboard.PasteboardType.string.rawValue)
    }
}
```

### 4. PasteService

**Purpose:** Handles paste injection via CGEvent. Replaces `fakeCommandV` + `addClipToPasteboard:`.

**Critical sequence:**
1. Capture `previousApp` at hotkey press (before bezel opens)
2. Write content to NSPasteboard
3. Call `previousApp.activate()`
4. Post CGEvent Cmd-V after brief yield (50ms)

```swift
@MainActor
class PasteService {
    var previousApp: NSRunningApplication?

    func captureFrontApp() {
        previousApp = NSWorkspace.shared.frontmostApplication
    }

    func paste(_ content: String, asPlainText: Bool = false) {
        let pb = NSPasteboard.general
        pb.clearContents()
        pb.setString(content, forType: .string)

        previousApp?.activate(options: .activateIgnoringOtherApps)

        DispatchQueue.main.asyncAfter(deadline: .now() + 0.05) {
            self.sendCommandV()
        }
    }

    private func sendCommandV() {
        let keyDown = CGEvent(keyboardEventSource: nil, virtualKey: 0x09, keyDown: true)!
        keyDown.flags = .maskCommand
        let keyUp = CGEvent(keyboardEventSource: nil, virtualKey: 0x09, keyDown: false)!
        keyDown.post(tap: .cghidEventTap)
        keyUp.post(tap: .cghidEventTap)
    }
}
```

### 5. HotkeyManager

**Purpose:** Registers and handles global keyboard shortcuts. Replaces SGHotKeysLib.

**Approach:** Use `KeyboardShortcuts` library (SPM). It wraps CGEventTap and provides both registration and a SwiftUI recorder view.

### 6. BezelWindowController

**Purpose:** Manages the non-activating bezel HUD panel. Replaces `BezelWindow` + related AppKit code.

**Key requirements:**
- `NSPanel` with `NSWindowStyleMaskNonactivatingPanel` — must NOT activate the app
- `panel.level = .screenSaver + 1` for visibility over fullscreen apps
- `panel.collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary]`
- Center on screen containing mouse cursor (not just primary screen)
- Hosts SwiftUI `BezelView` via `NSHostingView`

### 7. GistService

**Purpose:** Creates GitHub Gists from clipboard entries or snippets.

**Design:**
- `URLSession` async/await for GitHub REST API v3
- Personal Access Token stored in Keychain
- `Codable` models for Gist request/response
- No third-party HTTP client needed

### Snippet
```swift
@Model
class Snippet {
    var name: String
    var content: String
    var language: String?
    var category: String?
    var createdAt: Date
    var updatedAt: Date
}
```

### GistRecord
```swift
@Model
class GistRecord {
    var gistID: String
    var gistURL: String
    var filename: String
    var createdAt: Date
    // Links back to source clipping or snippet
}
```

**Key simplification:** The existing Flycut uses three separate `FlycutStore` objects (clippings, favorites, stashed). The Swift rewrite uses a single `Clipping` model with `isFavorite: Bool` and SwiftData predicates to filter. This eliminates the store-switching complexity.

## Data Flow

### Clipboard Capture Flow
```
Timer fires (0.5s) → PasteboardMonitor.checkPasteboard()
    → Checks changeCount, denylist, blockedCount
    → Returns (content, type) or nil
    → ClipboardController.handleNewClipping()
        → Deduplicates against in-memory list
        → Inserts at position 0
        → Persists via background ModelActor (debounced)
        → UI updates automatically via @Observable
```

### Paste Flow
```
Hotkey press → HotkeyManager callback
    → PasteService.captureFrontApp()
    → ClipboardController.showBezel()
    → User navigates with arrow keys (stackPosition changes)
    → User presses Enter/Return
    → PasteboardMonitor.blockNextChange()
    → PasteService.paste(selectedContent)
        → Write to NSPasteboard
        → Activate previousApp
        → Post CGEvent Cmd-V (50ms delay)
    → ClipboardController.hideBezel()
```

### Gist Sharing Flow
```
User selects "Share as Gist" → GistService.createGist()
    → Read PAT from Keychain
    → POST to api.github.com/gists
    → Save GistRecord to SwiftData
    → Copy Gist URL to clipboard
    → Show confirmation
```

## Suggested Build Order

Based on component dependencies:

```
Phase 1: Foundation
├── App shell (MenuBarExtra, Info.plist, entitlements)
├── SwiftData models (Clipping, Snippet, GistRecord)
├── AppSettings (@AppStorage preferences)
└── Accessibility checker

Phase 2: Core Engine
├── PasteboardMonitor (polling, denylist, block count)
├── PasteService (capture, write, CGEvent)
├── HotkeyManager (KeyboardShortcuts integration)
└── ClipboardController (state coordination)

Phase 3: UI
├── BezelWindowController (NSPanel + NSHostingView)
├── BezelView (SwiftUI clipping list, navigation)
├── MenuBarView (recent clippings, actions)
└── SettingsView (preferences, hotkey recorder)

Phase 4: Extended Features
├── Snippet editor (create, edit, syntax highlighting)
├── Snippet management (categories, search)
├── GistService (GitHub API, Keychain auth)
└── Gist UI (share sheet, history)
```
